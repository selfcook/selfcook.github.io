Block相关

Posted on 2020-04-18 | In code
需要理解的知识点：
  1. 常量存储在常量区，全局变量／静态变量存储在全局区/静态区。
    变量：1.1 基本数据类型／非oc对象一般存储在栈区，1.2 oc对象一般是存储在堆区
  2. oc对象有指针概念，指针变量存储在栈区，而指针存放的是对象在堆区的地址。所以我们是通过栈区的指针变量来索引堆中地址，从而访问对象的。
  3. Block不允许修改外部变量的值，这里所说的外部变量的值，指的是栈中指针的地址。要想在Block内修改“外部变量”的值，必须被__block修饰。

oc对象.png

1.基本数据类型(不被__block修饰)

没有被__block修饰的变量(栈区的a)被block持有（NSLog(@”a == %d”,a);）就会被复制一份（在堆区生成一个a），那么在外面修改变量的值（a = 33;栈区的a），对block内的变量（堆区的a）不起作用（输出 a == 100 不是33）。因为被复制后，这是两个同名但地址不一样的变量，所以对block中的a不影响。

2.oc对象(不被__block修饰)

原理同上，但是又有区别。区别在于：
结合第2个知识点，str是指针变量，是存储在栈区，”zw”才是被分配在堆区，而且block复制的是栈区的指针变量，而不是”zw”。复制后虽然栈区的str与堆区的str地址不一样，但是它两的存储内容是一样，都是”zw”的内存地址，指向的是同一单元对象。

另外str = @”once”;这里实质是修改栈区str的存储内容,将str重新指向堆中”once”，因为系统会在堆中开辟新的地址给”once”，并将str的内容由原先的”zw”内存地址换成”once”的内存地址。

2.oc对象(不被__block修饰).png

是不是很奇怪为什么这里可以修改mutableStr，不是不能修改外部变量吗？区别在于mutableStr.string = @”once”;上面说到复制后虽然栈区的mutableStr与堆区的mutableStr地址不一样，但是它两的存储内容是一样，指向的都是”zw”。所以对mutableStr.string进行操作，实际是对堆中的对象单元进行操作，这是可以的。是不是更好理解第三点了。

3.基本数据类型/oc对象(被__block修饰)

block修饰的变量block就会将“外部变量”在栈中的内存地址放到了堆中。无论是基本数据类型还是指针变量，都会被移到堆中，只会存在一个变量。这就是与没有被block修饰的区别所在。所以在block内修改，还是在block后面的外面修改都是修改堆中的变量。
